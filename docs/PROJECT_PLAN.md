## 1. 项目概述

本项目基于 SRS_v1.0 中定义的 REQ-00 至 REQ-11 功能需求，交付一个可运行的图书管理系统 MVP。系统采用**单体架构 + 前后端分离**模式，数据库使用 **PostgreSQL**。

**核心业务闭环**：

> 用户登录 → 读者注册 → 图书上架 → 图书检索 → 借书 → 查看在借清单 → 还书（含超期冻结与自动解冻）

## 3. 技术选型（基于 JDK 1.8）

由于当前 JDK 版本为 `1.8.0_401`，Spring Boot 3.x 系列要求 JDK 17 或更高版本，因此技术栈锁定在 **Spring Boot 2.x** 系列。

| 技术组件 | 选型版本 | 选型理由 |
| --- | --- | --- |
| JDK | 1.8.0_401 | 当前环境，稳定运行 |
| Spring Boot | 2.7.18 | 2.x 最终稳定版，完美支持 JDK 8，兼容至 JDK 21；社区资料丰富，排错成本低 |
| Spring Framework | 5.3.31+ | Spring Boot 2.7.18 自带依赖 |
| Spring Security | 5.7.x | Spring Boot 2.7.x 默认整合，与 JWT 深度集成 |
| Spring Cloud | 2021.0.x（如 2021.0.9） | Spring Boot 2.7.x 官方推荐搭配；MVP 虽不用微服务，但为后续架构演进预留扩展性 |
| ORM | MyBatis-Plus 3.5.x | 兼容 JDK 8，提供便捷的 CRUD 和分页支持 |
| 数据库 | PostgreSQL 14+ | SRS 明确指定 |
| JWT | jjwt 0.11.x | 轻量级，JDK 8 兼容 |
| 连接池 | HikariCP | Spring Boot 默认，性能优异 |
| 构建工具 | Maven 3.6+ | 稳定，依赖管理成熟 |
| 代码版本 | Git | 已初始化 GitHub 仓库 |

## 7. 风险登记册

| 风险编号 | 风险描述 | 概率 | 影响 | 应对措施 |
| --- | --- | --- | --- | --- |
| R-01 | Trae AI 免费版 Builder 调用次数有限（100次/月） | 中 | 高 | 优先用 DeepSeek 生成方案，Trae 仅用于最终代码落地 |

| R-03 | 1 周工期过紧，进度落后 | 中 | 高 | 每日站会跟踪进度；必要时砍掉 REQ-07/08/09/11（降级为 Should/Could） |
| R-04 | PostgreSQL 环境配置问题导致启动失败 | 低 | 中 | 提前准备 Docker 版的 PostgreSQL 快速启动脚本 |
| R-05 | JDK 8 与某些依赖版本不兼容 | 低 | 中 | 严格按照上述技术选型版本，不随意升级 |
