# EvoLib - 用户故事清单（MVP版）

> 基于图书馆馆长访谈整理，聚焦1周内可交付的最小可行产品。
> 本次更新补充了**前端交互描述**与**前端验收标准（GWT 格式：Given/When/Then）**，明确所有功能均通过浏览器界面操作，而非裸 API。

## 模块1：图书检索（读者端）

| ID | 用户故事 | 角色 | MoSCoW | 优先级说明 | 前端验收标准（GWT） |
| :--- | :--- | :--- | :--- | :--- | :--- |
| US-01 | 作为**普通读者**，我想要**在搜索框输入关键词并点击搜索按钮**，以便**查看图书列表**。 | 读者 | **M** | 核心功能，无此功能读者无法使用系统 | Given 读者已登录，When 在搜索框输入"Java"并点击搜索，Then 页面刷新显示包含"Java"的图书列表 |
| US-02 | 作为**普通读者**，我想要**查看图书的详细信息**（库存、位置、简介），以便**决定是否去借阅**。 | 读者 | **M** | 搜索结果必须有详情页支撑 | Given 读者已登录并查到图书，When 点击某图书条目，Then 页面展示该书的完整详情（总库存、当前可借数量、架位号、简介） |
