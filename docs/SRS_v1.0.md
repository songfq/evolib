## 1. 引言

### 1.1 目的

本规格说明书详细描述 EvoLib 图书馆管理 MVP 系统的功能需求、非功能需求、数据模型及接口定义，作为开发团队实现和测试团队验收的依据。

### 1.2 适用范围

本系统为 B/S 架构，面向普通读者（检索）、流通馆员（借还、读者管理）和系统管理员（图书上架/下架、参数配置）。MVP 版本部署于内网或云环境，支持主流浏览器（Chrome/Firefox/Edge）。

### 1.3 引用文档

- 《业务需求文档（BRD）》 v1.0
- 《用户故事清单（USER_STORIES.md）》

## 2. 整体功能架构

系统分为四个功能域：

- **读者服务域**：图书检索、借阅详情查看
- **借阅交易域**：借书、还书、在借清单
- **读者管理域**：注册、信息修改、密码重置（管理员）
- **图书管理域**：上架、下架

## 3. 功能需求（REQ 编号）

### 3.0 模块 M0：用户认证（所有角色）

| 需求编号 | 需求描述 | 输入 | 输出 | 验收标准 |
| --- | --- | --- | --- | --- |
| REQ-00 | 读者/馆员/管理员通过借阅证号和密码登录系统 | readerId（借阅证号），password（明文密码） | JWT token（有效期8小时），用户角色（ROLE_READER/ROLE_CIRCULATION/ROLE_ADMIN） | ① 密码验证使用bcrypt匹配；② 凭证错误时返回"用户名或密码错误"（不暴露具体原因）；③ 连续5次失败后锁定账户15分钟（MVP可简化）；④ token中包含readerId和role信息 |

### 3.1 模块 M1：图书检索（读者端）

| 需求编号 | 需求描述 | 输入 | 输出 | 验收标准 |
| --- | --- | --- | --- | --- |
| REQ-01 | 读者可按书名、作者或 ISBN 进行模糊/精确搜索图书 | 搜索关键词（文本），可选搜索类型（书名/作者/ISBN） | 图书列表（书名、作者、ISBN、当前可借数量、架位号） | 搜索响应 ≤ 2 秒；支持中文分词；无结果时显示友好提示 |
| REQ-02 | 读者可点击图书条目查看详情页 | 图书 ID（ISBN） | 图书详情：全部字段（书名、作者、ISBN、总库存、当前可借数量、架位号、简介/备注） | 详情页展示完整；库存数量与实时一致 |

### 3.2 模块 M2：借阅管理（馆员端）

| 需求编号 | 需求描述 | 输入 | 输出 | 验收标准 |
| --- | --- | --- | --- | --- |
| REQ-03 | 馆员通过读者 ID 和图书 ISBN 执行借书操作 | 读者 ID（借阅证号），图书 ISBN | 借阅成功回执（含借阅日期、应还日期），更新该读者「当前在借数量」+1，更新图书「当前可借数量」-1 | 校验读者是否存在、是否超期、是否达到借阅上限（默认 3 本）；校验图书是否存在、可借数量 > 0；任一失败返回明确错误码 |
| REQ-04 | 馆员通过图书 ISBN 执行还书操作 | 图书 ISBN（可选读者 ID 以加速定位） | 还书成功回执，更新该读者「当前在借数量」-1，更新图书「当前可借数量」+1 | 校验该图书确为当前读者所借；若该读者借阅记录中存在超期，还书后超期状态自动解除（可再次借书）。超期图书归还后，立即解除冻结，读者可当场再次借书（MVP 版本不设延迟解冻）。 |
| REQ-05 | 馆员可查看指定读者的当前在借清单 | 读者 ID | 在借图书列表（书名、ISBN、借阅日期、应还日期、是否超期） | 列表实时反映当前借阅状态；支持按超期状态筛选 |

### 3.3 模块 M3：读者管理（馆员端/管理员端）

| 需求编号 | 需求描述 | 输入 | 输出 | 验收标准 |
| --- | --- | --- | --- | --- |
| REQ-06 | 流通馆员可注册新读者（生成借阅证号） | 姓名（必填）、手机号（必填）、借阅证号（必填，系统唯一） | 读者账户创建成功，初始密码为默认（如身份证后六位或手机号后六位），借阅上限默认 3 本，当前在借数量 0 | 证号唯一性校验；手机号格式校验（11 位数字） |
| REQ-07 | 流通馆员可修改读者手机号 | 读者 ID + 新手机号 | 更新成功 | 权限校验：仅限馆员角色；新手机号格式合法 |
| REQ-08 | 系统管理员可重置读者密码 | 读者 ID | 密码重置为默认值，返回重置通知（仅管理员可见） | 仅管理员可操作；重置后读者可使用默认密码登录 |
| REQ-09 | 系统管理员可修改借阅规则参数（硬编码配置） | 配置文件（application.yml 或 .env）中的 max_borrow_days（默认 30 天）、max_borrow_count（默认 3 本） | 重启服务后生效 | 此为 MVP 临时方案，V1.1 版本将纳入管理后台 UI 配置。 |

### 3.4 模块 M4：图书管理（管理员端）

| 需求编号 | 需求描述 | 输入 | 输出 | 验收标准 |
| --- | --- | --- | --- | --- |
| REQ-10 | 系统管理员可上架新书 | 书名（必填）、作者（必填）、ISBN（必填，唯一）、总库存（必填，正整数）、架位号（建议必填）、简介（可选） | 图书记录创建成功，当前可借数量 = 总库存 | ISBN 唯一性校验；总库存 ≥ 1；架位号非空提醒（但不强制） |
| REQ-11 | 系统管理员可下架图书（逻辑删除） | 图书 ISBN | 图书「当前可借数量」置为 0，并标记为不可借（is_active = false） | 不影响历史借阅记录；检索时不再展示该书；若该书有在借记录，需先全部还书后才可下架 |

## 4. 非功能需求（NFR）

### 4.1 性能需求

| 编号 | 需求 | 指标 |
| --- | --- | --- |
| NFR-01 | 检索接口响应时间 | 95% 请求 ≤ 2 秒，并发 50 用户 |
| NFR-02 | 借书/还书交易响应时间 | 95% 请求 ≤ 1 秒 |
| NFR-03 | 系统吞吐量 | 支持至少 200 并发用户同时操作（含检索和交易混合场景） |
| NFR-04 | 数据库查询优化 | 图书检索表需对书名、作者、ISBN 建立索引，借阅记录表对读者 ID 和图书 ISBN 建立索引 |

### 4.2 安全与权限需求

| 编号 | 需求 | 描述 |
| --- | --- | --- |
| NFR-05 | 身份认证 | 所有接口（除登录外）需携带 JWT token；token 有效期为 8 小时 |
| NFR-06 | 角色权限控制（RBAC） | 三个角色：ROLE_READER（仅检索+个人借阅查看）、ROLE_CIRCULATION（借还+读者注册/修改手机号）、ROLE_ADMIN（上架/下架+重置密码+配置修改）。每个接口需校验角色 |
| NFR-07 | 数据加密 | 密码存储使用 bcrypt 或 PBKDF2 哈希，禁止明文存储 |
| NFR-08 | 日志审计 | 记录所有借还操作、读者注册、图书上架/下架的操作人、时间、IP 地址，日志保存不少于 90 天 |

### 4.3 可用性与可靠性

| 编号 | 需求 | 描述 |
| --- | --- | --- |
| NFR-09 | 系统可用性 | MVP 阶段目标 99%（计划内维护除外） |
| NFR-10 | 错误处理 | 所有业务异常（如超期拦截、库存不足）返回统一错误码和用户友好提示，不暴露系统堆栈 |
| NFR-11 | 数据备份 | 数据库每日自动备份，保留最近 7 天备份文件 |

### 4.4 易用性

| 编号 | 需求 | 描述 |
| --- | --- | --- |
| NFR-12 | 界面语言 | 全中文，所有提示信息清晰明确 |
| NFR-13 | 响应式设计 | 馆员端支持 PC 端；读者检索页同时适配平板和手机浏览器 |
| NFR-14 | 操作反馈 | 借书/还书操作成功后，页面显示成功信息并打印（或显示）回执内容（借阅日期、应还日期） |

## 5. 数据模型（核心实体）

### 5.3 借阅记录表（borrow_records）

| detail | JSON | | 附加数据 |
| ip_address | VARCHAR(45) | | 客户端 IP |
| created_at | TIMESTAMP | DEFAULT NOW() | |

## 6. 接口定义（API 规格）

### 6.1 通用约定

- **基础路径**：`/api/v1`
- **认证方式**：Bearer Token（JWT），登录接口除外
- **数据格式**：JSON（请求/响应）
- **错误码规范**：统一使用 HTTP 状态码 + 业务错误码（见附录）

### 6.2 核心接口列表

#### 6.2.0 登录接口

```http
POST /auth/login
```

请求体：
```json
{
  "readerId": "R20240001",
  "password": "123456"
}
```

响应：
```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "role": "ROLE_CIRCULATION"
}
```

> 馆员和管理员用同一套登录接口，系统根据 readers.role 字段返回对应权限

#### 6.2.1 图书检索接口

```http
GET /books/search
```

Query 参数：

- `keyword`: string（必填）
- `type`: enum('title', 'author', 'isbn')，默认 'title'
- `page`: int，默认 1
- `size`: int，默认 10

响应：

```json
{
  "code": 0,
  "data": {
    "total": 25,
    "list": [
      {
        "isbn": "978-7-111-12345-6",
        "title": "深入理解计算机系统",
        "author": "Randal E. Bryant",
        "available_stock": 3,
        "shelf_location": "A区-3排-5架"
      }
    ]
  }
}
```

#### 6.2.2 图书详情接口

```http
GET /books/{isbn}
```

响应：图书完整对象（含总库存、简介等）

#### 6.2.3 借书接口

```http
POST /borrow-records
```

请求体：

```json
{
  "readerId": "R20240001",
  "isbn": "978-7-111-12345-6"
}
```

响应（成功）：

```json
{
  "code": 0,
  "data": {
    "borrowId": 1001,
    "borrowDate": "2026-07-09",
    "dueDate": "2026-08-08"
  }
}
```

错误示例：

```json
{
  "code": 4001,
  "message": "读者存在超期未还图书，无法借书"
}
```

#### 6.2.4 还书接口

```http
POST /borrow-records/{recordId}/return
```

请求体：

```json
{
  "isbn": "978-7-111-12345-6",
  "readerId": "R20240001"
}
```

响应：

```json
{
  "code": 0,
  "data": {
    "returnDate": "2026-07-09",
    "overdueDays": 0
  }
}
```

#### 6.2.5 读者在借清单

```http
GET /readers/{readerId}/borrows
```

响应：借阅记录列表（含图书信息、应还日期、超期标记）

#### 6.2.6 注册读者（馆员）

```http
POST /readers
```

请求体：

```json
{
  "readerId": "R20240088",
  "name": "张明",
  "phone": "13800001111"
}
```

响应：读者完整信息（含默认密码提示）

#### 6.2.7 修改手机号

```http
PUT /readers/{readerId}/phone
```

请求体：

```json
{
  "phone": "13900002222"
}
```

#### 6.2.8 重置密码（管理员）

```http
PUT /admin/readers/{readerId}/reset-password
```

响应：

```json
{
  "newPassword": "默认密码"
}
```

#### 6.2.9 上架图书（管理员）

```http
POST /admin/books
```

请求体：

```json
{
  "isbn": "978-7-302-12345-6",
  "title": "算法导论",
  "author": "Thomas H. Cormen",
  "totalStock": 5,
  "shelfLocation": "B区-1排-2架",
  "description": "经典计算机教材"
}
```

#### 6.2.10 下架图书（管理员）

```http
DELETE /admin/books/{isbn}
```

> 逻辑删除，标记 `is_active = false`

## 7. 附录

### 7.1 业务错误码表

| 错误码 | 说明 |
| --- | --- |
| 4001 | 读者超期未还，禁止借书 |
| 4002 | 读者已达到借阅上限 |
| 4003 | 图书可借数量为 0 |
| 4004 | 读者不存在 |
| 4005 | 图书不存在 |
| 4006 | 该书未被该读者借阅（还书异常） |
| 4007 | 读者 ID 或 ISBN 格式不合法 |
| 4008 | 图书已下架，不可操作 |
| 4009 | 读者已有在借该图书（重复借阅） |

### 7.2 MVP 配置参数（硬编码）

| 参数名 | 默认值 | 说明 |
| --- | --- | --- |
| library.max.borrow.days | 30 | 借阅天数 |
| library.max.borrow.count | 3 | 读者最大借阅本数 |
| library.default.password | 手机号后六位 | 注册时默认密码 |


## 8. 用户界面需求（UI 需求）

### 8.1 页面清单与功能描述

#### 8.1.1 未认证页面

| 页面编号 | 页面名称 | 访问角色 | 页面核心内容描述 | 对应后端接口 |
| --- | --- | --- | --- | --- |
| P-01 | 登录页 | 未认证用户 | 包含“借阅证号”输入框、“密码”输入框、“登录”按钮。登录失败时在按钮上方显示红色错误提示（内容为“用户名或密码错误”）。 | `POST /auth/login` |

#### 8.1.2 读者端页面

| 页面编号 | 页面名称 | 访问角色 | 页面核心内容描述 | 对应后端接口 |
| --- | --- | --- | --- | --- |
| P-02 | 图书检索首页 | 读者 | 顶部显示当前登录读者姓名及“退出”按钮。主体区域包含：搜索条件区（关键词输入框 + 搜索类型下拉框（书名/作者/ISBN）+ “搜索”按钮）；搜索结果展示区（以列表/表格形式展示图书：书名、作者、ISBN、可借数量、架位号）；底部分页控件（每页默认10条）。每条数据旁设有“查看详情”链接/按钮。 | `GET /books/search` |
| P-03 | 图书详情页 | 读者 | 展示图书全部信息：书名、作者、ISBN、总库存、可借数量、架位号、简介。页面底部设有“返回检索页”链接。 | `GET /books/{isbn}` |

#### 8.1.3 馆员端页面

| 页面编号 | 页面名称 | 访问角色 | 页面核心内容描述 | 对应后端接口 |

### 8.2 页面交互通用规则（业务层面）

| 规则编号 | 规则描述（纯业务/行为描述，不含技术实现） |
| --- | --- |
| UI-01 | **未登录拦截**：用户直接访问任何功能页面时，系统自动跳转至登录页。 |
| UI-02 | **权限拦截**：读者无法看到/进入馆员和管理员的功能页面；馆员无法看到/进入管理员的功能页面。若用户通过URL直接访问越权页面，系统应返回“无权限”提示并引导至其主页。 |
| UI-03 | **表单基础校验**：所有提交按钮点击前，系统需在前端提醒用户填写必填项（如“借阅证号不能为空”），手机号格式须为11位数字。 |
| UI-04 | **操作反馈**：所有增、删、改操作（借书、还书、注册、上架、下架、重置密码）完成后，系统必须给出明确的“成功”或“失败”的弹窗/页面提示。失败提示需直接显示业务错误码对应的中文含义（参考错误码表），不得显示技术异常堆栈。 |
| UI-05 | **数据刷新**：在借书成功后，页面需自动刷新当前读者的在借清单，无需馆员手动刷新页面。 |
| UI-06 | **二次确认**：下架（删除）操作必须触发二次确认弹窗，防止误操作。 |

> 本 SRS 经开发、测试、业务三方评审通过后，作为项目开发与验收的唯一依据。
