# 2026-07-13 — 登录后页面跳转机制详解

---

## 一、核心疑问

**用户登录后，是怎么跳转到 AppLayout.vue 界面的？**

答案：**用户登录后并没有直接跳转到 AppLayout.vue**，而是每个页面组件内部引用了 AppLayout 组件。

---

## 二、完整跳转流程

### 2.1 流程图

```
用户在登录页输入账号密码 → 点击登录按钮
    │
    ├─► ① LoginView.vue: handleLogin()
    │       │
    │       └─► auth.login(readerId, password)
    │               │
    │               ├─► ② auth.js: login()
    │               │       │
    │               │       ├─► api.post('/auth/login', { readerId, password })
    │               │       │       │
    │               │       │       └─► 后端验证账号密码，返回 Token
    │               │       │
    │               │       ├─► 保存 Token、角色、用户名到 localStorage
    │               │       │
    │               │       └─► router.push(homePath.value)
    │               │               │
    │               │               └─► 根据角色跳转到对应首页
    │               │                       │
    │               │                       ├─► ROLE_READER → /reader/search
    │               │                       ├─► ROLE_CIRCULATION → /circulation/borrow
    │               │                       └─► ROLE_ADMIN → /admin/add-book
    │               │
    │               └─► ③ Vue Router 路由匹配
    │                       │
    │                       ├─► 匹配到路由配置
    │                       │       │
    │                       │       └─► /reader/search → BookSearchView.vue
    │                       │
    │                       └─► ④ <router-view> 渲染页面组件
    │                               │
    │                               └─► BookSearchView.vue 内部包含 <AppLayout>
    │                                       │
    │                                       └─► ⑤ AppLayout 被渲染（侧边栏 + 头部 + 内容区）
    │
    └─► 用户看到带有左右结构的主界面
```

### 2.2 阶段详解

#### 阶段 1：点击登录按钮

**代码位置**：[LoginView.vue](file:///e:/ai_fq/evolib/code/evolib-frontend/src/views/LoginView.vue#L28-L45)

```javascript
async function handleLogin() {
  errorMsg.value = '';
  if (!readerId.value.trim() || !password.value.trim()) {
    errorMsg.value = '请输入读者ID和密码';
    return;
  }
  loading.value = true;
  try {
    const resp = await auth.login(readerId.value.trim(), password.value);
    if (resp.code !== 0) {
      errorMsg.value = resp.message || '登录失败';
    }
  } catch {
    errorMsg.value = '网络异常，请稍后重试';
  } finally {
    loading.value = false;
  }
}
```

**发生了什么**：
1. 验证用户输入
2. 调用 `auth.login()` 执行登录
3. 根据返回结果显示成功或失败提示

#### 阶段 2：执行登录逻辑

**代码位置**：[auth.js](file:///e:/ai_fq/evolib/code/evolib-frontend/src/stores/auth.js#L23-L35)

```javascript
async function login(readerId, password) {
  const resp = await api.post('/auth/login', { readerId, password });
  if (resp.code === 0) {
    token.value = resp.data.token;
    role.value = resp.data.role;
    readerName.value = resp.data.name || readerId;
    localStorage.setItem('evolib_token', resp.data.token);
    localStorage.setItem('evolib_role', resp.data.role);
    localStorage.setItem('evolib_name', readerName.value);
    router.push(homePath.value);
  }
  return resp;
}
```

**发生了什么**：
1. 调用后端登录接口 `/api/v1/auth/login`
2. 登录成功后，保存 Token、角色、用户名到：
   - Pinia store（内存）：`token.value`, `role.value`, `readerName.value`
   - localStorage（本地存储）：页面刷新后仍能保持登录状态
3. 执行 `router.push(homePath.value)` 跳转

**homePath 的计算**：

```javascript
const homePath = computed(() => ({
  'ROLE_READER': '/reader/search',
  'ROLE_CIRCULATION': '/circulation/borrow',
  'ROLE_ADMIN': '/admin/add-book',
}[role.value] || '/login'));
```

根据用户角色，跳转到不同的首页。

#### 阶段 3：路由匹配

**代码位置**：[router/index.js](file:///e:/ai_fq/evolib/code/evolib-frontend/src/router/index.js#L4-L16)

```javascript
const routes = [
  { path: '/login', name: 'Login', component: () => import('@/views/LoginView.vue'), meta: { auth: false } },
  { path: '/reader/search', name: 'BookSearch', component: () => import('@/views/reader/BookSearchView.vue') },
  { path: '/circulation/borrow', name: 'Borrow', component: () => import('@/views/circulation/BorrowView.vue') },
  { path: '/admin/add-book', name: 'AddBook', component: () => import('@/views/admin/AddBookView.vue') },
  ...
];
```

**发生了什么**：
1. `router.push('/reader/search')` 触发路由导航
2. Vue Router 遍历 `routes` 数组，找到匹配的路由
3. `/reader/search` 匹配到第 6 条路由配置
4. 动态加载 `BookSearchView.vue` 组件

#### 阶段 4：页面渲染

**代码位置**：[App.vue](file:///e:/ai_fq/evolib/code/evolib-frontend/src/App.vue)

```vue
<template>
  <router-view />
</template>
```

**发生了什么**：
1. `<router-view>` 根据当前路由渲染对应的组件
2. 当前路由匹配到 `BookSearchView.vue`，所以渲染为 `<BookSearchView />`

#### 阶段 5：AppLayout 被渲染

**代码位置**：[BookSearchView.vue](file:///e:/ai_fq/evolib/code/evolib-frontend/src/views/reader/BookSearchView.vue#L1-L17)

```vue
<template>
  <AppLayout>
    <div class="page-container">
      <h2 class="page-title">图书检索</h2>
      ...
    </div>
  </AppLayout>
</template>

<script setup>
import AppLayout from '@/components/layout/AppLayout.vue';
...
</script>
```

**关键发现**：
- BookSearchView.vue **内部引用了 AppLayout 组件**
- 当 BookSearchView 被渲染时，AppLayout 也随之被渲染
- 页面内容通过 `<slot />` 传入 AppLayout

## 三、为什么不直接跳转到 AppLayout？

### 3.1 设计模式：布局组件作为容器

```
传统方式（不推荐）：
┌─────────────────────────────────────┐
│ 路由配置：                          │
│ /app → AppLayout.vue               │
│ /app/search → BookSearchView.vue   │
│ （嵌套路由）                        │
└─────────────────────────────────────┘

当前方式（推荐）：
┌─────────────────────────────────────┐
│ 路由配置：                          │
│ /reader/search → BookSearchView.vue │
│ （BookSearchView 内部引用 AppLayout）│
└─────────────────────────────────────┘
```

### 3.2 两种方式对比

| 方式 | 优点 | 缺点 |
| --- | --- | --- |
| **嵌套路由** | 布局统一管理，路由结构清晰 | 需要配置嵌套路由，代码稍复杂 |
| **组件引用** | 简单直观，每个页面独立 | 每个页面都要 import AppLayout |

### 3.3 当前项目选择组件引用的原因

1. **简单直观**：对于 MVP 项目，组件引用方式更简单
2. **灵活性高**：不同页面可以选择不同的布局
3. **易于理解**：初学者更容易理解这种方式

## 六、关键知识点总结

| 问题 | 答案 |
| --- | --- |
| **登录后直接跳转到 AppLayout 吗？** | ❌ 不直接跳转，而是跳转到具体页面 |
| **AppLayout 是怎么显示的？** | 每个页面组件内部引用了 AppLayout，页面渲染时 AppLayout 随之渲染 |
| **为什么这样设计？** | 简单直观，灵活性高，易于理解 |
| **登录后跳转的目标是什么？** | 根据用户角色跳转到对应的首页路径（如 `/reader/search`） |
| **页面内容如何显示在右侧？** | 通过 AppLayout 的 `<slot />` 插槽机制 |
| **登录状态如何保持？** | Token 保存在 localStorage，页面刷新后仍能保持登录 |

## 七、扩展：嵌套路由方式（可选）

如果后续需要更规范的路由结构，可以改为嵌套路由方式：

```javascript
// router/index.js
const routes = [
  { path: '/login', name: 'Login', component: () => import('@/views/LoginView.vue'), meta: { auth: false } },
  {
    path: '/',
    component: () => import('@/components/layout/AppLayout.vue'),
    children: [
      { path: 'reader/search', name: 'BookSearch', component: () => import('@/views/reader/BookSearchView.vue') },
      { path: 'circulation/borrow', name: 'Borrow', component: () => import('@/views/circulation/BorrowView.vue') },
      { path: 'admin/add-book', name: 'AddBook', component: () => import('@/views/admin/AddBookView.vue') },
      // ... 其他页面
    ]
  },
];
```

```vue
<!-- AppLayout.vue -->
<template>
  <div class="app-layout">
    <AppSidebar />
    <AppHeader />
    <main class="app-layout__main">
      <router-view />  <!-- 嵌套路由的出口 -->
    </main>
  </div>
</template>
```

这样修改后：
- 每个页面组件不再需要引用 AppLayout
- 布局统一由路由配置管理
- AppLayout 中的 `<router-view />` 作为嵌套路由的出口
