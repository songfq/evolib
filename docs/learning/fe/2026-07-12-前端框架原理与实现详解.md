## 二、核心组件详解

### 2.1 Vue 3 启动原理

**这是什么？** Vue 3 是一个渐进式 JavaScript 框架，通过响应式系统实现数据驱动视图。

**为什么这么做？** 传统的 DOM 操作繁琐且易出错，Vue 的响应式系统让数据变化自动更新视图。

**凭什么这么做？**

```javascript
// main.js — 应用入口
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'
import pinia from './stores'
import './styles/theme.css'
import './styles/common.css'

const app = createApp(App)
app.use(router)    // 安装路由插件
app.use(pinia)     // 安装状态管理插件
app.mount('#app')  // 将应用挂载到 DOM 元素
```

启动流程：
1. **createApp(App)**：创建 Vue 应用实例
2. **app.use(router)**：安装 Vue Router 插件
3. **app.use(pinia)**：安装 Pinia 状态管理插件
4. **app.mount('#app')**：将应用挂载到 index.html 的 `#app` 元素

### 2.2 组件化开发 — Vue 单文件组件

**这是什么？** Vue 单文件组件（SFC）将模板、脚本、样式封装在一个 `.vue` 文件中。

**为什么这么做？** 组件化开发提高代码复用性，便于维护和测试。

**核心要点：**

```vue
<!-- LoginView.vue -->
<template>
  <div class="login-container">
    <div class="login-form">
      <h2>图书馆管理系统</h2>
      <EvoInput
        v-model="form.readerId"
        label="借阅证号"
        placeholder="请输入借阅证号"
      />
      <EvoInput
        v-model="form.password"
        label="密码"
        type="password"
        placeholder="请输入密码"
      />
      <EvoButton type="primary" @click="handleLogin" :loading="loading">
        登录
      </EvoButton>
    </div>
  </div>
</template>

<script setup>
import { ref, reactive } from 'vue'
import { ElMessage } from 'element-plus'
import { useAuthStore } from '@/stores/auth'
import EvoInput from '@/components/EvoInput.vue'
import EvoButton from '@/components/EvoButton.vue'

const authStore = useAuthStore()
const loading = ref(false)
const form = reactive({
  readerId: '',
  password: ''
})

async function handleLogin() {
  if (!form.readerId || !form.password) {
    ElMessage.error('请填写借阅证号和密码')
    return
  }
  
  loading.value = true
  try {
    await authStore.login(form.readerId, form.password)
    ElMessage.success('登录成功')
    router.push(authStore.homePath)
  } catch (error) {
    ElMessage.error(error.message)
  } finally {
    loading.value = false
  }
}
</script>

<style scoped>
.login-container {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
  background: var(--color-bg-primary);
}

.login-form {
  width: 400px;
  padding: 32px;
  background: var(--color-bg-white);
  border-radius: var(--radius-lg);
  box-shadow: var(--shadow-lg);
}
</style>
```

组件三要素：
- **template**：HTML 模板，使用 Vue 指令（v-model、v-if、v-for 等）
- **script setup**：JavaScript 逻辑，使用 Composition API（ref、reactive、computed 等）
- **style scoped**：CSS 样式，scoped 确保样式只作用于当前组件

### 2.3 响应式系统 — ref 和 reactive

**这是什么？** Vue 3 的响应式系统通过 `ref` 和 `reactive` 将普通数据变成响应式数据。

**为什么这么做？** 数据变化时自动更新视图，无需手动操作 DOM。

**核心要点：**

```javascript
import { ref, reactive, computed } from 'vue'

// ref — 用于基本类型（String、Number、Boolean）
const count = ref(0)
console.log(count.value)  // 读取值：0
count.value++             // 修改值：1

// reactive — 用于对象和数组
const form = reactive({
  readerId: '',
  password: ''
})
console.log(form.readerId)  // 读取值：''
form.readerId = 'R001'      // 修改值

// computed — 计算属性，依赖变化时自动重新计算
const fullName = computed(() => {
  return form.firstName + ' ' + form.lastName
})
```

响应式原理：Vue 3 使用 **Proxy** 代理对象，拦截属性的读取和写入操作：
1. **track**：读取属性时，记录依赖关系
2. **trigger**：写入属性时，触发依赖更新

### 2.4 Vue Router — 路由管理

**这是什么？** Vue Router 是 Vue 官方的路由管理器，用于构建单页应用（SPA）。

**为什么这么做？** 单页应用需要在不刷新页面的情况下切换视图。

**核心要点：**

```javascript
// router/index.js
import { createRouter, createWebHistory } from 'vue-router'

const routes = [
  {
    path: '/login',
    name: 'Login',
    component: () => import('@/views/LoginView.vue'),
    meta: { requiresAuth: false }
  },
  {
    path: '/reader/search',
    name: 'BookSearch',
    component: () => import('@/views/reader/BookSearchView.vue'),
    meta: { requiresAuth: true, roles: ['READER', 'CIRCULATION', 'ADMIN'] }
  },
  {
    path: '/circulation/borrow',
    name: 'Borrow',
    component: () => import('@/views/circulation/BorrowView.vue'),
    meta: { requiresAuth: true, roles: ['CIRCULATION', 'ADMIN'] }
  },
  {
    path: '/admin/add-book',
    name: 'AddBook',
    component: () => import('@/views/admin/AddBookView.vue'),
    meta: { requiresAuth: true, roles: ['ADMIN'] }
  },
  {
    path: '/:pathMatch(.*)*',
    redirect: '/login'
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router
```

路由配置要点：
- **懒加载**：`() => import('...')` 按需加载组件，减小首屏体积
- **meta**：路由元信息，用于权限控制
- **redirect**：路由重定向，匹配不到时跳转

### 2.5 路由守卫 — beforeEach

**这是什么？** 路由守卫在路由跳转前后执行，用于权限控制和页面访问控制。

**为什么这么做？** 防止未登录用户访问受限页面，防止低权限用户访问高权限页面。

**核心要点：**

```javascript
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()
  
  // 公开页面（如登录页）直接放行
  if (!to.meta.requiresAuth) {
    next()
    return
  }
  
  // 未登录用户跳转到登录页
  if (!authStore.token) {
    next('/login')
    return
  }
  
  // 已登录用户检查角色权限
  const roles = to.meta.roles
  if (roles && !roles.includes(authStore.role)) {
    // 角色不匹配，跳转到对应角色的首页
    next(authStore.homePath)
    return
  }
  
  // 权限通过，继续跳转
  next()
})
```

守卫执行顺序：
1. **全局前置守卫**（beforeEach）→ 权限校验
2. **路由独享守卫**（beforeEnter）
3. **组件内守卫**（beforeRouteEnter）
4. **全局解析守卫**（beforeResolve）
5. **全局后置钩子**（afterEach）→ 页面切换后的操作

### 2.6 Pinia — 状态管理

**这是什么？** Pinia 是 Vue 3 的官方状态管理库，用于管理应用的全局状态。

**为什么这么做？** 多个组件共享状态时，需要一个集中式的状态管理方案。

**核心要点：**

```javascript
// stores/auth.js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import { api } from '@/utils/api'

export const useAuthStore = defineStore('auth', () => {
  // 状态
  const token = ref(localStorage.getItem('token') || '')
  const role = ref('')
  const readerName = ref('')
  const readerId = ref('')
  
  // 计算属性
  const isLoggedIn = computed(() => !!token.value)
  
  const homePath = computed(() => {
    switch (role.value) {
      case 'ROLE_ADMIN':
        return '/admin/add-book'
      case 'ROLE_CIRCULATION':
        return '/circulation/borrow'
      case 'ROLE_READER':
        return '/reader/search'
      default:
        return '/login'
    }
  })
  
  // 动作
  async function login(readerId, password) {
    const response = await api.post('/auth/login', { readerId, password })
    token.value = response.data.token
    role.value = response.data.role
    readerName.value = response.data.readerName
    readerId.value = response.data.readerId
    
    localStorage.setItem('token', token.value)
    localStorage.setItem('role', role.value)
    localStorage.setItem('readerName', readerName.value)
    localStorage.setItem('readerId', readerId.value)
  }
  
  function logout() {
    token.value = ''
    role.value = ''
    readerName.value = ''
    readerId.value = ''
    
    localStorage.removeItem('token')
    localStorage.removeItem('role')
    localStorage.removeItem('readerName')
    localStorage.removeItem('readerId')
  }
  
  function hasRole(targetRole) {
    return role.value === targetRole
  }
  
  return {
    token,
    role,
    readerName,
    readerId,
    isLoggedIn,
    homePath,
    login,
    logout,
    hasRole
  }
})
```

Pinia 优势：
- **Composition API**：使用 ref、computed、函数定义状态和操作
- **持久化**：配合 localStorage 保存登录状态
- **模块化**：每个 Store 独立管理相关状态

### 2.7 Axios 封装 — API 请求

**这是什么？** Axios 是一个基于 Promise 的 HTTP 客户端，用于发送 API 请求。

**为什么这么做？** 统一处理请求/响应，自动添加认证 Token，统一错误处理。

**核心要点：**

```javascript
// utils/api.js
import axios from 'axios'
import { useAuthStore } from '@/stores/auth'
import router from '@/router'

const api = axios.create({
  baseURL: '/api/v1',
  timeout: 10000
})

// 请求拦截器 — 自动添加 JWT Token
api.interceptors.request.use(
  config => {
    const authStore = useAuthStore()
    if (authStore.token) {
      config.headers.Authorization = `Bearer ${authStore.token}`
    }
    return config
  },
  error => {
    return Promise.reject(error)
  }
)

// 响应拦截器 — 统一处理响应和错误
api.interceptors.response.use(
  response => {
    // 统一解包：后端返回 {code, message, data}
    const res = response.data
    if (res.code !== 0) {
      // 业务错误，抛出异常
      return Promise.reject(new Error(res.message))
    }
    return res
  },
  error => {
    if (error.response) {
      switch (error.response.status) {
        case 401:
          // Token 过期或无效，强制登出
          const authStore = useAuthStore()
          authStore.logout()
          router.push('/login')
          break
        case 403:
          // 无权限
          return Promise.reject(new Error('无权限访问'))
        case 500:
          // 服务器错误
          return Promise.reject(new Error('服务器内部错误'))
      }
    }
    return Promise.reject(error)
  }
)

export { api }
```

拦截器工作流程：
1. **请求拦截器**：在发送请求前添加 Token
2. **发送请求**：通过 Vite 代理转发到后端
3. **响应拦截器**：统一解包响应，处理错误状态码

### 2.8 Vite 代理 — 跨域解决方案

**这是什么？** Vite 代理将前端 API 请求转发到后端服务器，解决跨域问题。

**为什么这么做？** 浏览器同源策略禁止跨域请求，代理让前端请求看起来像是来自同一域名。

**核心要点：**

```javascript
// vite.config.js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { resolve } from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src')
    }
  },
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        rewrite: path => path.replace(/^\/api/, '')
      }
    }
  }
})
```

代理流程：
1. 前端发送请求 `http://localhost:3000/api/v1/auth/login`
2. Vite 代理转发到 `http://localhost:8080/api/v1/auth/login`
3. 后端处理请求并返回响应
4. Vite 代理将响应返回给前端

## 三、关键技术点总结

| 技术点 | 是什么 | 为什么用 | 凭什么这么做 |
| --- | --- | --- | --- |
| Vue 3 | 渐进式框架 | 响应式数据绑定，组件化开发 | Proxy 代理实现响应式 |
| Vite | 构建工具 | 极速冷启动，HMR 热更新 | ESM 原生模块 + Rollup |
| Vue Router | 路由管理器 | 单页应用路由控制 | 虚拟 DOM 实现无刷新切换 |
| Pinia | 状态管理 | 集中式管理全局状态 | Composition API + 模块化 |
| Axios | HTTP 客户端 | 拦截器支持，Promise API | 统一请求/响应处理 |
| ref/reactive | 响应式数据 | 数据变化自动更新视图 | Proxy 拦截属性读写 |
| v-model | 双向绑定 | 表单数据自动同步 | 语法糖：v-bind + v-on |
| scoped style | 组件样式隔离 | 样式不污染全局 | CSS 属性选择器 |
