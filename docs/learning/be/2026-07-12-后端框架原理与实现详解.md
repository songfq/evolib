## 二、核心组件详解

### 2.1 Spring Boot 启动原理

**这是什么？** Spring Boot 是一个快速开发框架，通过自动配置（AutoConfiguration）减少样板代码。

**为什么这么做？** 传统 Spring 需要大量 XML 配置，Spring Boot 通过 `@SpringBootApplication` 注解一键启动。

**凭什么这么做？**

```java
@SpringBootApplication
public class EvolibBackendApplication {
    public static void main(String[] args) {
        SpringApplication.run(EvolibBackendApplication.class, args);
    }
}
```

启动流程：
1. **扫描组件**：`@SpringBootApplication` = `@ComponentScan` + `@EnableAutoConfiguration` + `@SpringBootConfiguration`
2. **自动配置**：Spring Boot 根据 `pom.xml` 中的依赖自动配置 Bean（如数据库连接池、JPA、Redis等）
3. **启动内嵌 Tomcat**：默认端口 8080，无需单独部署 Tomcat

### 2.2 Controller 层 — 请求入口

**这是什么？** Controller 是 HTTP 请求的入口，负责接收请求、参数校验、调用 Service、返回响应。

**为什么这么做？** 遵循 MVC 分层原则，Controller 只做请求适配，不包含业务逻辑。

**核心要点：**

```java
@RestController
@RequestMapping("/api/v1/auth")
public class AuthController {
    
    @Autowired
    private AuthService authService;
    
    @PostMapping("/login")
    public Result<LoginResponse> login(@Valid @RequestBody LoginRequest request) {
        LoginResponse response = authService.login(request);
        return Result.ok(response);
    }
}
```

关键注解：
- `@RestController`：= `@Controller` + `@ResponseBody`，返回 JSON 而不是视图
- `@RequestMapping`：定义路由前缀
- `@PostMapping`：处理 POST 请求
- `@Valid`：触发参数校验（配合 `@NotNull`、`@Size` 等注解）
- `@RequestBody`：将 JSON 请求体映射到 Java 对象

### 2.3 Service 层 — 业务逻辑核心

**这是什么？** Service 层包含所有业务逻辑，是整个系统的核心。

**为什么这么做？** 业务逻辑集中在 Service，便于单元测试和代码复用。

**核心要点：**

```java
@Service
public class BorrowServiceImpl implements BorrowService {
    
    @Autowired
    private BorrowRecordMapper borrowRecordMapper;
    
    @Autowired
    private BookMapper bookMapper;
    
    @Autowired
    private ReaderMapper readerMapper;
    
    @Transactional(rollbackFor = Exception.class)
    @Override
    public BorrowRecordDTO borrow(BorrowRequest request) {
        // Step 1: 校验读者存在性
        Reader reader = readerMapper.selectById(request.getReaderId());
        if (reader == null) {
            throw new BusinessException(ErrorCode.READER_NOT_FOUND);
        }
        
        // Step 2: 检查超期状态
        long overdueCount = borrowRecordMapper.countOverdueByReaderId(request.getReaderId());
        if (overdueCount > 0) {
            throw new BusinessException(ErrorCode.HAS_OVERDUE);
        }
        
        // Step 3: 检查借阅上限
        if (reader.getCurrentBorrowCount() >= reader.getMaxBorrowCount()) {
            throw new BusinessException(ErrorCode.EXCEED_MAX_BORROW);
        }
        
        // Step 4: 校验图书存在性
        Book book = bookMapper.selectById(request.getIsbn());
        if (book == null || !book.getIsActive()) {
            throw new BusinessException(ErrorCode.BOOK_NOT_FOUND);
        }
        
        // Step 5: 扣库存（行锁防超卖）
        int updateCount = bookMapper.decreaseStock(request.getIsbn());
        if (updateCount == 0) {
            throw new BusinessException(ErrorCode.STOCK_INSUFFICIENT);
        }
        
        // Step 6: 检查重复借阅
        BorrowRecord existing = borrowRecordMapper.selectByReaderAndBook(
            request.getReaderId(), request.getIsbn());
        if (existing != null && "BORROWED".equals(existing.getStatus())) {
            throw new BusinessException(ErrorCode.DUPLICATE_BORROW);
        }
        
        // Step 7: 创建借阅记录
        BorrowRecord record = new BorrowRecord();
        record.setReaderId(request.getReaderId());
        record.setIsbn(request.getIsbn());
        record.setBorrowDate(LocalDate.now());
        record.setDueDate(LocalDate.now().plusDays(maxBorrowDays));
        record.setStatus("BORROWED");
        borrowRecordMapper.insert(record);
        
        // Step 8: 更新读者借阅计数
        readerMapper.increaseBorrowCount(request.getReaderId());
        
        // Step 9: 写入审计日志
        auditLogMapper.insert(AuditLog.builder()
            .operatorId(request.getReaderId())
            .action("BORROW")
            .targetType("BOOK")
            .targetId(request.getIsbn())
            .detail("读者[" + request.getReaderId() + "]借阅图书[" + request.getIsbn() + "]")
            .build());
        
        return convertToDTO(record, book, reader);
    }
}
```

关键技术点：
- `@Service`：声明这是一个服务层 Bean
- `@Transactional`：声明式事务，所有 SQL 要么全成功要么全回滚
- `@Autowired`：依赖注入，Spring 自动装配 Bean
- 异常驱动：通过 `BusinessException` 返回精确错误码

### 2.4 Mapper 层 — 数据访问

**这是什么？** Mapper 层（也叫 DAO 层）负责与数据库交互，执行 SQL 语句。

**为什么这么做？** 使用 MyBatis-Plus 简化 CRUD 操作，无需手写 SQL。

**核心要点：**

```java
@Mapper
public interface BookMapper extends BaseMapper<Book> {
    
    @Update("UPDATE books SET available_stock = available_stock - 1 WHERE isbn = #{isbn} AND available_stock > 0")
    int decreaseStock(String isbn);
    
    @Select("SELECT * FROM books WHERE is_active = true AND " +
            "(title LIKE CONCAT('%', #{keyword}, '%') OR " +
            "author LIKE CONCAT('%', #{keyword}, '%') OR " +
            "isbn LIKE CONCAT('%', #{keyword}, '%'))")
    IPage<Book> search(Page<Book> page, @Param("keyword") String keyword);
}
```

关键技术点：
- `@Mapper`：声明这是一个 MyBatis Mapper 接口
- `BaseMapper<T>`：MyBatis-Plus 提供的通用 Mapper，包含 CRUD 方法
- 自定义 SQL：使用 `@Select`、`@Update`、`@Insert`、`@Delete` 注解
- 分页：`IPage<T>` 自动处理分页查询

### 2.5 Spring Security — 安全守卫

**这是什么？** Spring Security 是一个功能强大的安全框架，提供认证（Authentication）和授权（Authorization）。

**为什么这么做？** 安全是企业级应用的必备功能，Spring Security 提供了成熟的解决方案。

**核心要点：**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Autowired
    private JwtAuthenticationFilter jwtAuthenticationFilter;
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf().disable()
            .sessionManagement().sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            .and()
            .authorizeHttpRequests()
                .antMatchers("/api/v1/auth/login").permitAll()
                .antMatchers("/api/v1/books/**").hasAnyRole("READER", "CIRCULATION", "ADMIN")
                .antMatchers("/api/v1/borrow-records/**", "/api/v1/readers/**").hasAnyRole("CIRCULATION", "ADMIN")
                .antMatchers("/api/v1/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            .and()
            .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);
        
        return http.build();
    }
    
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

安全流程：
1. **请求到达** → 进入 Spring Security 过滤器链
2. **JwtAuthenticationFilter** → 从请求头提取 Token，解析并校验
3. **SecurityContext** → 将用户信息放入上下文
4. **授权校验** → 根据 URL 路径检查角色权限
5. **Controller** → 请求到达目标方法

### 2.6 JWT — 无状态认证

**这是什么？** JWT（JSON Web Token）是一种无状态的认证方案，将用户信息编码到 Token 中。

**为什么这么做？** 无状态认证不需要服务器保存会话，适合前后端分离架构和水平扩展。

**凭什么这么做？**

```java
@Component
public class JwtTokenProvider {
    
    @Value("${jwt.secret}")
    private String secret;
    
    @Value("${jwt.expiration}")
    private long expiration;
    
    public String generateToken(String readerId, String role) {
        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + expiration);
        
        return Jwts.builder()
            .setSubject(readerId)
            .claim("role", role)
            .setIssuedAt(now)
            .setExpiration(expiryDate)
            .signWith(SignatureAlgorithm.HS512, secret)
            .compact();
    }
    
    public String getReaderIdFromToken(String token) {
        Claims claims = Jwts.parser()
            .setSigningKey(secret)
            .parseClaimsJws(token)
            .getBody();
        
        return claims.getSubject();
    }
    
    public boolean validateToken(String token) {
        try {
            Jwts.parser().setSigningKey(secret).parseClaimsJws(token);
            return true;
        } catch (JwtException | IllegalArgumentException e) {
            return false;
        }
    }
}
```

JWT 结构：
```
eyJhbGciOiJIUzUxMiIsInR5cCI6IkpXVCJ9.
eyJzdWIiOiJSMDAxIiwicm9sZSI6IlJPTEVfUkVBRERFUiIsImlhdCI6MTcyMDg0MzQyNywiZXhwIjoxNzIwODQ2MDI3fQ.
xxx_signature_xxx
```
- **Header**：声明算法和类型
- **Payload**：存储用户信息（readerId、role、过期时间）
- **Signature**：签名，防止篡改

### 2.7 统一响应体 — Result\<T\>

**这是什么？** 统一的响应格式，所有接口返回相同结构。

**为什么这么做？** 前端处理方便，统一错误处理。

```java
public class Result<T> {
    private Integer code;
    private String message;
    private T data;
    
    private Result(Integer code, String message, T data) {
        this.code = code;
        this.message = message;
        this.data = data;
    }
    
    public static <T> Result<T> ok(T data) {
        return new Result<>(0, "success", data);
    }
    
    public static <T> Result<T> fail(Integer code, String message) {
        return new Result<>(code, message, null);
    }
}
```

### 2.8 全局异常处理 — GlobalExceptionHandler

**这是什么？** 统一捕获所有异常，返回标准错误格式。

**为什么这么做？** 避免暴露技术堆栈，统一错误响应格式。

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(BusinessException.class)
    public Result<Void> handleBusinessException(BusinessException e) {
        return Result.fail(e.getErrorCode().getCode(), e.getMessage());
    }
    
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<Void> handleValidationException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldErrors().stream()
            .map(FieldError::getDefaultMessage)
            .collect(Collectors.joining(", "));
        return Result.fail(ErrorCode.PARAM_VALID_ERROR.getCode(), message);
    }
    
    @ExceptionHandler(AccessDeniedException.class)
    public Result<Void> handleAccessDeniedException(AccessDeniedException e) {
        return Result.fail(403, "无权限访问");
    }
    
    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        return Result.fail(500, "服务器内部错误");
    }
}
```

## 三、完整请求处理流程

以登录请求为例：

```
POST /api/v1/auth/login
Body: {"readerId": "R001", "password": "123456"}

┌──────────────────────────────────────────────────────────────┐
│ 1. 请求到达 Tomcat 8080 端口                                   │
│ 2. Spring MVC 映射到 AuthController.login()                   │
│ 3. @Valid 校验参数（readerId、password 不为空）                 │
│ 4. AuthService.login() 执行：                                 │
│    ├─ ReaderMapper.selectById("R001") 查询读者               │
│    ├─ BCryptPasswordEncoder.matches() 比对密码                │
│    ├─ JwtTokenProvider.generateToken() 生成 Token            │
│    └─ 返回 LoginResponse {token, role, readerName}           │
│ 5. Result.ok(response) 包装响应                               │
│ 6. 返回 JSON: {"code":0, "message":"success", "data":{...}}  │
└──────────────────────────────────────────────────────────────┘
```

## 四、关键技术点总结

| 技术点 | 是什么 | 为什么用 | 凭什么这么做 |
| --- | --- | --- | --- |
| Spring Boot | 快速开发框架 | 约定优于配置，减少样板代码 | 自动配置机制 + 内嵌服务器 |
| 三层架构 | Controller/Service/Mapper | 职责分离，便于测试和维护 | 单一职责原则 |
| @Transactional | 声明式事务 | 多条 SQL 原子操作 | Spring AOP 自动管理事务 |
| JWT | 无状态认证 | 适合前后端分离，支持水平扩展 | Token 自包含用户信息 |
| Spring Security | 安全框架 | 成熟的认证授权体系 | 过滤器链机制 |
| MyBatis-Plus | ORM框架 | 简化 CRUD，减少重复代码 | BaseMapper 通用方法 |
| 统一响应体 | Result\<T\> | 前端处理统一，便于错误处理 | 所有接口返回相同结构 |
| 全局异常处理 | GlobalExceptionHandler | 统一错误响应，隐藏技术细节 | @RestControllerAdvice 拦截异常 |
