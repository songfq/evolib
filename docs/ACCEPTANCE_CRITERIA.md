# EvoLib - 验收标准（GWT格式）

## 故事1：图书检索（US-01）

| 场景 | Given（前置条件） | When（触发动作） | Then（预期结果） |
| :--- | :--- | :--- | :--- |
| 按书名搜索 | 数据库中存在名为"Java核心技术"的图书 | 读者在搜索框输入"Java"并点击搜索 | 返回包含"Java核心技术"的列表 |
| 按作者搜索 | 数据库中存在"余华"写的《活着》 | 读者在搜索框输入"余华"并点击搜索 | 返回《活着》及余华的其他作品 |
| ISBN精确搜索 | 数据库中存在ISBN=9787111111111的图书 | 读者输入完整ISBN | 精确返回该书的详情 |
| 无结果搜索 | 数据库中不存在"xxxxxx" | 读者输入"xxxxxx" | 返回空列表并提示"未找到相关图书" |

## 故事2：借书（US-03）

| 场景 | Given（前置条件） | When（触发动作） | Then（预期结果） |
| :--- | :--- | :--- | :--- |
| 正常借书 | 读者"张三"可借额度未满，图书"Java核心技术"可借数量>0 | 馆员输入读者ID+图书ISBN并点击"借书" | ① 生成借阅记录<br>② 图书可借数量-1<br>③ 读者在借数量+1<br>④ 返回借书成功 |
| 读者超期冻结 | 读者"李四"名下有超期未还图书 | 馆员尝试为李四借书 | 系统拦截，提示"该读者存在超期图书，无法借书" |
| 库存不足 | 图书"数据结构"可借数量=0 | 读者尝试借该书 | 系统拦截，提示"该图书已全部借出" |
| 借阅上限已满 | 读者"王五"已借3本（上限3本） | 王五尝试借第4本 | 系统拦截，提示"已达借阅上限" |

## 故事3：还书（US-04）

| 场景 | Given（前置条件） | When（触发动作） | Then（预期结果） |
| :--- | :--- | :--- | :--- |
| 正常还书 | 读者"张三"持有《Java核心技术》未还 | 馆员输入图书ISBN点击"还书" | ① 借阅记录标记为"已还"<br>② 图书可借数量+1<br>③ 读者在借数量-1<br>④ 返回还书成功 |
| 还书时自动计算超期天数 | 图书应还日期为2026-07-07，当前日期为2026-07-09 | 馆员执行还书操作 | 系统自动计算超期天数=2，并记录到借阅记录中（仅记录，不罚款） |

## 故事4：读者注册（US-06）

| 场景 | Given（前置条件） | When（触发动作） | Then（预期结果） |
| :--- | :--- | :--- | :--- |
| 正常注册 | 读者ID未被占用 | 馆员录入读者信息（ID、姓名、手机号） | 读者记录创建成功，默认密码为系统预设值 |
| 重复注册 | 读者ID已存在 | 馆员录入重复ID | 系统提示"读者ID已存在，请核对" |
| 手机号格式校验 | 手机号长度不为11位 | 馆员录入错误格式手机号 | 系统提示"手机号格式不正确" |

## 故事5：图书上架（US-10）

| 场景 | Given（前置条件） | When（触发动作） | Then（预期结果） |
| :--- | :--- | :--- | :--- |
| 新书上架 | ISBN不存在于数据库 | 管理员录入完整图书信息 | 图书记录创建成功，可借数量=总库存 |
| ISBN重复 | ISBN已存在于数据库 | 管理员录入重复ISBN | 系统提示"该ISBN已存在，请勿重复录入" |
| 必填项校验 | 书名或作者为空 | 管理员提交不完整表单 | 系统提示"书名/作者为必填项" |
