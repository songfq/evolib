# integration-test — 集成测试规范

## 目标

定义 EvoLib 项目的集成测试策略：**核心业务流程端到端验证 + RBAC 权限矩阵验证 + 错误码全覆盖**。

---

## 测试方式

| 方式 | 用途 | 适用场景 |
| --- | --- | --- |
| **Postman Collection** 手动/自动化 | API 级集成测试 | 开发阶段快速验证接口 |
| **Spring Boot Test** + TestRestTemplate | 自动化集成测试 | CI/CD 流水线中自动执行 |

MVP 阶段优先使用 **Postman Collection**（`assets/postman-collection.json`），后续 CI 阶段可转为 Spring Boot Test。

---

## 一、核心业务流程测试

### 场景 1：完整借还书闭环

```
用户：张三（读者，R001）  →  登录系统
用户：李四（馆员，R002）  →  登录系统
用户：王五（管理员，R003）→  登录系统

管理员 → 上架图书《Java 编程思想》(ISBN: 978-7-111-1, 库存 3)
读者   → 搜索 "Java" → 看到《Java 编程思想》
读者   → 点击查看详情 → 显示库存 3 本

馆员   → 注册新读者「赵六」(R004)
馆员   → 赵六借书《Java 编程思想》→ 借书成功，显示应还日期
馆员   → 查赵六在借清单 → 显示 1 本书
馆员   → 赵六还书《Java 编程思想》→ 还书成功，超期天数 0

馆员   → 再次借书《Java 编程思想》给赵六 → 成功（库存足够）

管理员 → 下架《Java 编程思想》→ 因赵六持有在借本 → 提示「请先归还全部在借图书」
馆员   → 赵六还书《Java 编程思想》→ 成功
管理员 → 下架《Java 编程思想》→ 成功
```

### 场景 2：超期冻结与自动解冻

```
（手动修改数据库：将赵六的某条借阅记录的 borrow_date 改为 31 天前）
馆员 → 赵六借书另一本书 → 后端返回 4001「读者存在超期未还图书，无法借书」
馆员 → 赵六还书那本超期的书 → 还书成功，超期天数 1 天，提示「已解除借阅冻结」
馆员 → 赵六再次借书 → 成功（冻结已自动解除）
```

### 场景 3：库存不足并发

```
（手动设置《Java 编程思想》available_stock = 1）
同时执行 3 个借书该书的请求 → 
  请求 1：借书成功
  请求 2：返回 4003「图书可借数量不足」
  请求 3：返回 4003「图书可借数量不足」
```

## 三、错误码全覆盖清单

逐个触发以下错误，验证后端返回正确的 `{code, message}`，前端显示中文提示：

| 错误码 | 触发方式 | 预期后端响应 |
| --- | --- | --- |
| 4001 | 借书时读者有超期记录（手动修改 borrow_records） | `{"code":4001,"message":"读者存在超期未还图书，无法借书"}` |
| 4002 | 读者已借 3 本书时再借一本 | `{"code":4002,"message":"读者已达到借阅上限"}` |
| 4003 | 借书时 available_stock = 0 | `{"code":4003,"message":"图书可借数量不足"}` |
| 4004 | 用不存在的 readerId 借书 | `{"code":4004,"message":"读者不存在"}` |
| 4005 | 用不存在的 isbn 借书 | `{"code":4005,"message":"图书不存在"}` |
| 4006 | 还书时用从未借过该书的读者 | `{"code":4006,"message":"该书未被该读者借阅"}` |
| 4007 | 借书请求缺少 readerId 或 isbn | `{"code":4007,"message":"参数格式不合法"}` |
| 4008 | 对已下架的书执行操作 | `{"code":4008,"message":"图书已下架，不可操作"}` |
| 4009 | 借书时读者已借过同一本书 | `{"code":4009,"message":"读者已借阅此书"}` |

## 四、性能基线

| 指标 | 目标值 | 测量方式 |
| --- | --- | --- |
| 搜索接口响应时间 | 95% ≤ 2s，并发 50 | `curl -w "%{time_total}"` 或 Postman Runner |
| 借/还书响应时间 | 95% ≤ 1s | 同上 |
| 并发借书无超卖 | 多线程同时借最后一本，仅 1 人成功 | 场景 3 的并发测试 |

## 五、集成测试 Check List

每次发布前，对照以下清单执行：

- [ ] 场景 1：张三查书 → 李四借书 → 李四还书 完整闭环通过
- [ ] 场景 2：超期冻结 → 还书解冻 → 再次借书通过
- [ ] 场景 3：库存不足并发无超卖
- [ ] RBAC：3 种角色各自不能越权
- [ ] 错误码：4001~4009 全部触发并返回正确中文消息
- [ ] 搜索性能：50 并发 ≤ 2s
- [ ] 登录：正确密码成功 / 错误密码返回「用户名或密码错误」
- [ ] Token 过期：后端返回 401 → 前端跳转登录页
