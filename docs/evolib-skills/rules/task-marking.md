# 任务标记规则

## 1. 规则目标

规范 tasklist.md 中任务完成状态的标记行为，避免错误地将未完成任务标记为已完成，确保任务状态的准确性和可信度。

## 2. 核心原则

### 2.1 精确标记原则

**只标记用户明确指定的任务，不进行任何扩散或推断。**

- 当用户说"标记 Phase 1 完成"时，只标记 Phase 1 的任务
- 当用户说"标记 T-DB-01 完成"时，只标记 T-DB-01
- **禁止**根据代码文件存在就自动标记任务完成
- **禁止**根据其他任务完成就推断关联任务完成

### 2.2 用户意图优先原则

用户的指令是唯一的标记依据：

- 用户说"标记 Phase 1" → 标记 Phase 1 下所有任务
- 用户说"标记 T-BE-01" → 只标记 T-BE-01
- 用户说"标记数据库完成" → 根据上下文判断对应哪个任务/Phase

### 2.3 状态一致性原则

每次标记任务后，同步更新统计概览中的"已完成"和"待完成"数量。

## 3. 标记操作规范

### 3.1 标记格式

在任务名称前添加 ✅ 标记：

```markdown
| **T-DB-01** | **✅ 执行建库建表 DDL** | ... |
```

### 3.2 统计更新

标记完成后，同步更新顶部统计概览：

```markdown
| **已完成** | **X** |
| **待完成** | **Y** |
```

其中 X + Y = 总任务数（50）

### 3.3 回滚操作

当标记错误时，需要：
1. 移除错误添加的 ✅ 标记
2. 恢复统计概览中的数量

## 4. 意图识别指南

### 4.1 用户指令类型

| 用户指令 | 识别结果 | 操作 |
| --- | --- | --- |
| "标记 Phase 1 完成" | 标记 Phase 1 下所有任务 | 标记 T-DB-01, T-DB-02, T-DB-03 |
| "标记数据库完成" | 标记 Phase 1（数据库）下所有任务 | 标记 T-DB-01, T-DB-02, T-DB-03 |
| "标记 T-DB-01 完成" | 只标记 T-DB-01 | 只标记 T-DB-01 |
| "更新 tasklist" | 根据上下文判断，通常指更新当前正在完成的任务 | 需要进一步确认用户意图 |

### 4.2 模糊指令处理

当用户指令不够明确时：

1. **优先确认**：询问用户具体要标记哪个任务/Phase
2. **不要猜测**：避免根据代码存在推断任务完成
3. **保持谨慎**：宁可少标记，不可多标记

### 4.3 禁止行为

| 行为 | 说明 |
| --- | --- |
| ❌ 批量推断 | 不要看到代码文件就自动标记所有相关任务 |
| ❌ 关联推断 | 不要因为 Phase 1 完成就自动标记 Phase 2 |
| ❌ 过度标记 | 不要标记用户没有提到的任何任务 |

## 5. 决策流程

```
用户请求标记任务
    │
    ├── 解析用户指令中的任务标识
    │       │
    │       ├── 明确指定了任务 ID（如 T-DB-01）→ 标记该任务
    │       ├── 指定了 Phase（如 Phase 1）→ 标记该 Phase 下所有任务
    │       └── 模糊描述（如"数据库完成"）→ 询问确认
    │
    ├── 执行标记操作
    │       │
    │       └── 在任务名称前添加 ✅
    │
    ├── 更新统计概览
    │       │
    │       └── 更新已完成和待完成数量
    │
    └── 返回确认信息
            │
            └── 告诉用户标记了哪些任务
```

## 6. 示例

### 6.1 正确操作

**用户**："标记 T-DB-01 完成"

**操作**：
1. 在 tasklist.md 中找到 T-DB-01
2. 在任务名称前添加 ✅
3. 更新统计概览（已完成 +1）
4. 返回："已标记 T-DB-01（执行建库建表 DDL）为完成"

### 6.2 错误操作

**用户**："数据库表已经创建"

**错误操作**：标记 Phase 1 下所有任务 + Phase 2 + Phase 3...

**正确操作**：询问用户具体要标记哪些任务，或者根据上下文只标记数据库相关任务

## 7. 注意事项

1. **每次标记前确认**：确保完全理解用户意图
2. **只修改 tasklist.md**：不要修改其他文件
3. **保持格式一致**：使用统一的 ✅ 标记格式
4. **及时回滚**：发现标记错误时立即回滚
